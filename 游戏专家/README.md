# memory 目录说明

## 两类记忆文件

关卡大师的记忆系统分两层：

| 文件 | 位置 | 用途 | 读取时机 |
|------|------|------|----------|
| `MEMORY.md` | Agent 根目录 | **长期记忆**：稳定偏好 + 近期任务摘要 + 跨会话结论 | 每次会话启动时必读 |
| `memory/YYYY-MM-DD.md` | memory/ 目录 | **每日日志**：单次会话详细记录 | 启动时读最新一条 |

---

## 目录结构

```
memory/
├── _TEMPLATE.md             ← daily 日志模板（复制后改文件名使用）
├── YYYY-MM-DD.md            ← daily，按日期（每次会话结束后写入）
├── weekly/YYYY-Www.md       ← weekly，按 ISO 周
└── archive/{YYYY}-W{ww}/    ← archive，周报生成后挪入
```

---

## MEMORY.md 写入规则（长期记忆）

**写入时机**：每次会话结束，将以下内容同步写入根目录 `MEMORY.md`：

- **长期偏好**：用户明确表达的稳定偏好才写入（如"只看移动版数值""分析要细到波次"）；临时要求不写
- **近期任务摘要**：一行一条，格式为 `YYYY-MM-DD：<游戏>+<关卡/任务>+<核心结论>`，保留最近 10 条，超出时删最旧一条
- **已记录的游戏知识**：跨会话多次用到的结论才写入，避免重复解释；写入时注明游戏名和版本

**不写入 MEMORY.md 的内容**：单次偶发的策略讨论、用户临时要求、skill 调用原始结果（这些放 daily log）

---

## daily 日志写入规则（YYYY-MM-DD.md）

**写入时机**：每次会话结束后写入，文件名 = 当天日期

**使用方式**：复制 `_TEMPLATE.md`，改文件名为 `YYYY-MM-DD.md`，填写各字段

**字段说明**：

| 字段 | 必填 | 说明 |
|------|------|------|
| frontmatter date/game/level/mode | 是 | 便于日后快速定位 |
| 会话摘要 | 是 | 今日任务 + 关键结论，2-5 条 |
| 事实记录 | 按需 | 长期有效的分析结论（也会同步到 MEMORY.md） |
| 偏好更新 | 按需 | 用户新表达的偏好（也会同步到 MEMORY.md） |
| Skill 调用记录 | 按需 | 本次用了哪些 skill，结果是否有效 |
| 待办 | 按需 | 未完成事项，下次会话跟进 |

---

## archive 规则

- **weekly**：用户要求时手动生成周报到 `weekly/YYYY-Www.md`
- **标记**：周报生成后，给对应 daily 顶部 frontmatter 填入 `summarized_in: weekly/YYYY-Www.md`
- **归档**：填标记后将 daily 文件移入 `archive/{YYYY}-W{ww}/`
- **未标记的跳过**，不静默吞

---

## 删除语义

用户说"删除记录"按**存档到 archive**处理，不物理删除。数据可恢复性优先。
