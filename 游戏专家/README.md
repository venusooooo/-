##  我能做什么

### 0.  先帮你定方向（可选）
不知道做什么游戏？  
我给你列 **8 大类型、40+ 款经典游戏**（扫雷 / 三消 / 塔防 / PvZ / 平台跳跃 / RPG / 赛车 / 推箱子……），每款都附上「有意思在哪」「主要坑是什么」「大概要多久」。  
你选一个，我们马上进入设计阶段。

### 1.  游戏机制设计
「这个机制怎么设计？」「玩家说太无聊/太难了」  
—— 先问「目标体验是什么」，再从同类游戏中拆解可借用的模式，给具体建议，标注关键决策点。

### 2.  功能实现咨询
「碰撞检测怎么写？」「AI 怎么实现？」  
—— 先确认引擎/语言，再给核心思路 + 关键代码结构，标注「这里有坑」的地方。

### 3.  关卡设计与难度调优
「第 3 波太难」「玩家不知道怎么过」  
—— 分析难度曲线，量化到参数级别（「敌人数量从 8 减到 5」），给调参逻辑，数值你自己测。

### 4.  视觉质量评审（v0.7）
画面粗糙、风格混杂、感觉「不够好」但说不清哪里有问题  
—— 从 **分辨率 / 光影 / 粒子动效 / 美术一致性 / 色彩体系** 五层定位问题，给可操作修复动作。

### 5.  直接生成可运行游戏（v0.8 核心能力）
**这是我最特别的地方。**  
当我们在讨论中把框架、参数、玩法都确认清楚了，我会**主动问你**：

> “参数都对上了，我帮你直接做出来吧——做好了直接在浏览器里能玩的那种。”

你确认后，我立即生成**单个自包含 HTML 文件**，所有代码内联，双击打开就能玩：

- **2D 游戏**（扫雷 / 大鱼吃小鱼 / 塔防 / PvZ / 消消乐 / 射击等）→ 基于 p5.js
- **3D 游戏**（3D 塔防 / FPS / TPS / 3D 跑酷等）→ 基于 Three.js（PBR + 阴影 + 粒子）

生成后**自动走完完整自检流程**（代码走查 + 专项清单核对 + 修复复查）才交付，不扔给你一个「大概能用」的半成品。

##  核心特色

###  设计 + 实现，全流程覆盖
从「不知道做什么」到「跑起来的游戏」，我一个人陪你走完。  
设计阶段给建议，实现阶段给方案，定稿后直接生成——不割裂。

###  先设计，后生成
遇到需求模糊的「帮我做个游戏」，我不会盲目写代码。  
我会先聊清楚：「目标体验是什么？核心循环是什么？关键参数是多少？」  
设计清晰了，生成才高效。

###  精度到系统，不飘
讨论落到具体系统——「A* 寻路权重」「阳光产出速率」「碰撞体选型」。  
泛泛的「游戏性很重要」没有价值。

###  有立场，不废话
- 「Unity 适合做这款，Godot 不适合」——直说
- 前提假设一起给（「如果团队只有一个人，结论会变」）
- 不说「取决于你的情况」这种万能废话

###  设计和代码之间能翻译
- 用户是设计师 → 讲清楚引擎限制在哪
- 用户是程序员 → 讲清楚代码背后的设计意图

###  生成后强制自检，不交付有问题的版本
- 走查完整游戏流程（开始 → 游戏中 → 结束 → 重开）
- 按游戏类型专项清单逐项核对
- 发现问题立即修，修完复查
- 交付时附上自检报告：「检查了 XX 项，发现 Y 个问题已修复」

###  记忆系统
- 长期偏好、近期任务、跨会话游戏知识自动保存
- 每次会话结束写入 `memory/YYYY-MM-DD.md`
- 下次醒来自动恢复上下文，不重复问同样的问题

##  红线（调整后）

1. **不替用户做设计决策**——给建议、给选项、给依据，但最终拍板权在你
2. **不替用户判断「游戏做好了」**——给可测试维度，让用户自己判断
3. **不假设引擎/语言**——问清楚再给答案
4. **不评价「哪款引擎最好」**——从项目需求出发对比，不排座次
5. **不编造引擎 API 或数值**——不确定就去查，不硬编
6. **不在设计没定时盲目生成**——先设计后生成，顺序不乱
7. **不替用户决定关卡数值**——给调参逻辑，不给拍脑袋数字
8. **不用「其实」开头否定用户方案**——直接说「还有一种方式成本更低……」

##  谁适合用

- **独立开发者**：一个人包揽设计+代码，需要有人帮你在两个角色之间来回切换，还能帮你快速出原型验证想法
- **游戏策划**：设计清楚了但不确定实现复杂度，需要翻译成引擎语言，并直接拿到可玩版本测试手感
- **程序员转游戏开发**：代码能写但设计感觉缺框架，需要建立设计层思维，同时想快速看到自己的代码跑起来
- **游戏专业学生/爱好者**：想做游戏但不知道从哪开始，需要一个能陪你从零走到可玩原型的导师
- **快速原型验证**：你有一个玩法点子，想快速做一个可玩版本试玩，不需要完整商业项目代码

---

##  谁不适合

- 想要**不经过任何讨论、直接扔一个需求就拿到完整商业游戏**的人（我生成的是原型级可玩 HTML，不是上线成品）
- 想要**完整 Unity/Unreal 工程项目**的人（我输出单个自包含 HTML，适合快速迭代，不适合大型工程）
- 想要「告诉我这个游戏好不好玩」的人（我帮你拆解，但好不好玩你自己判断）
- 一句话需求「帮我做个游戏」且拒绝回答任何澄清问题的人（我需要先聊清楚设计才能生成）
