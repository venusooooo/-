# SKILL-game-catalog.md - 游戏推荐与对话引导

> 供流程H（游戏类型选择与推荐）调用。
> 包含两部分：**推荐数据库**（八大类型 × 4-5款游戏）+ **对话引导话术规范**（推荐时怎么说）。

---

## 一、推荐数据库

### 类型菜单（展示给用户时附氛围描述）

```
你想做哪类游戏？

  一、休闲轻度小游戏 ——上手秒懂，单局短，随时随地都能玩
  二、射击类         ——子弹满天飞，手速越快越爽
  三、塔防策略类     ——布防守线，看精心设计的防御体系发挥效果
  四、策略经营类     ——种地建城搞钱，看自己的系统慢慢运转
  五、横版闯关动作   ——跑跳打，经典不过时
  六、角色扮演 RPG   ——养角色探世界，打怪升级
  七、竞速 / 体育    ——追速度，超对手，刷新记录
  八、解谜密室       ——动脑解题，「啊原来如此！」的那种满足感

有哪个感觉对的吗？或者说不确定也行，告诉我你喜欢玩什么类型的游戏，我帮你找方向。
```

---

### 一、休闲轻度小游戏

**定位**：操作简单、单局短、适合碎片时间；核心乐趣在于重复可玩性和微目标满足感。

| 游戏 | 一句话说明 | 核心机制 | 开发复杂度 |
|------|-----------|---------|-----------|
| **扫雷** | 逻辑推理+概率判断的经典益智游戏 | 二维网格 + 地雷生成算法 + 递归展开 | ⭐⭐ |
| **2048** | 合并方块数字的数字益智游戏 | 二维数组滑动 + 数值合并逻辑 | ⭐⭐ |
| **消消乐（三消）** | 三个相同颜色/形状连成一线消除 | 网格匹配检测 + 下落填充 + 连消判断 | ⭐⭐⭐ |
| **贪吃蛇** | 控制蛇吃食物并越长越难 | 队列数据结构 + 方向控制 + 边界/自碰检测 | ⭐⭐ |
| **打砖块** | 用球弹射打击上方砖块 | 物理反弹计算 + 碰撞检测 + 砖块生命值 | ⭐⭐ |

**推荐首选**：扫雷（最经典）、三消（可玩性最高）

---

### 二、射击类

**定位**：快速反应 + 精准操作；核心爽点是消灭敌人的即时反馈和子弹射击手感。

| 游戏 | 一句话说明 | 核心机制 | 开发复杂度 |
|------|-----------|---------|-----------|
| **太空侵略者** | 固定炮台横向移动射击入侵外星人 | 子弹生成/移动 + 敌人阵列移动 + 碰撞检测 | ⭐⭐ |
| **飞机大战（弹幕射击）** | 垂直滚动地图，躲避弹幕消灭敌机 | 弹幕生成算法 + 速度差值 + 敌人波次 | ⭐⭐⭐ |
| **顶视角双摇杆射击** | 俯视视角自由移动+独立射击方向 | 双摇杆输入解析 + 全向射击 + 敌人AI | ⭐⭐⭐ |
| **第一人称射击（FPS）原型** | 第一视角移动+瞄准射击 | 3D场景 + 射线检测（Raycast）+ 视角控制 | ⭐⭐⭐⭐ |
| **弹珠台/弹射消除** | 弹射小球消灭方块/敌人 | 物理弹射轨迹预测 + 角度计算 + 连锁消除 | ⭐⭐⭐ |

**推荐首选**：飞机大战（最平衡）、太空侵略者（最快出原型）

---

### 三、塔防策略类

**定位**：策略布置防御 + 经济管理；核心乐趣是看精心布置的防御体系发挥效果。

| 游戏 | 一句话说明 | 核心机制 | 开发复杂度 |
|------|-----------|---------|-----------|
| **经典直线塔防** | 敌人沿固定路径前进，沿路放塔 | A*寻路 + 炮塔攻击范围 + 敌人波次系统 | ⭐⭐⭐ |
| **植物大战僵尸（PvZ类）** | 在格子里放置植物对抗横向推进的僵尸 | 格子系统 + 资源产出/消耗（阳光）+ 碰撞检测 | ⭐⭐⭐ |
| **迷宫建造塔防** | 玩家可以自由建造墙/塔来改变敌人路径 | 动态寻路（实时A*）+ 路径可达性验证 | ⭐⭐⭐⭐ |
| **TD with 技能** | 基础塔防 + 主角可以主动施放技能 | 塔防框架 + 主角控制 + 技能冷却/特效 | ⭐⭐⭐⭐ |

**推荐首选**：PvZ类（设计感最强）、经典直线塔防（技术难度最平衡）

---

### 四、策略经营类

**定位**：资源管理 + 建造/发展循环；核心乐趣是看自己的系统运转和长期规划的满足感。

| 游戏 | 一句话说明 | 核心机制 | 开发复杂度 |
|------|-----------|---------|-----------|
| **农场经营** | 种地/养殖/卖出，资金循环 | 时间系统 + 格子种植 + 市场价格波动 | ⭐⭐⭐ |
| **城市建造** | 放置建筑，管理人口/资源/满意度 | 格子放置 + 多资源平衡 + 人口增长曲线 | ⭐⭐⭐⭐ |
| **回合制策略** | 棋盘上的单位互相攻击，基于回合 | 六边形/方格棋盘 + 回合顺序 + 单位属性 | ⭐⭐⭐⭐ |
| **放置挂机游戏** | 离线产出 + 升级 + 无尽进度 | 数值增长曲线 + 离线收益计算 + 解锁系统 | ⭐⭐⭐ |
| **卡牌组建（Deck Building）** | 不断从卡池获取/优化卡组，逐关推进 | 卡牌抽取算法 + 效果堆叠 + 随机事件 | ⭐⭐⭐⭐ |

**推荐首选**：放置挂机（最快出可玩原型）、农场经营（设计空间最大）

---

### 五、横版闯关动作

**定位**：跳跃/战斗/移动的横版视角；核心乐趣是操作手感和关卡探索。

| 游戏 | 一句话说明 | 核心机制 | 开发复杂度 |
|------|-----------|---------|-----------|
| **平台跳跃（Platformer）** | 控制角色跳跃穿越平台、避开陷阱 | 重力系统 + 跳跃物理 + 碰撞体 + 关卡编辑 | ⭐⭐⭐ |
| **横版清版动作** | 向右推进消灭画面内的所有敌人 | 近战攻击判定 + 敌人AI（巡逻/追击）+ 连招系统 | ⭐⭐⭐⭐ |
| **银河战士类（Metroidvania）** | 大地图探索 + 能力解锁 + 回头路 | 地图管理 + 能力门控 + 存档点 | ⭐⭐⭐⭐⭐ |
| **无尽横版跑酷** | 自动向右奔跑，跳跃/滑铲避开障碍 | 程序化生成障碍 + 速度加速 + 分数系统 | ⭐⭐ |

**推荐首选**：无尽跑酷（最快出原型）、平台跳跃（经典且可扩展）

---

### 六、角色扮演 RPG

**定位**：角色成长 + 剧情/探索；核心乐趣是角色变强和世界探索的沉浸感。

| 游戏 | 一句话说明 | 核心机制 | 开发复杂度 |
|------|-----------|---------|-----------|
| **俯视角 ARPG** | 俯视视角自由移动+实时战斗（类《塞尔达》） | 碰撞检测 + 攻击判定帧 + 道具/武器系统 | ⭐⭐⭐⭐ |
| **回合制 RPG** | 遭遇敌人进入回合战斗界面（类《口袋妖怪》） | 回合顺序算法 + 技能系统 + 属性克制 | ⭐⭐⭐⭐ |
| **Roguelike/Roguelite** | 随机地图 + 死亡从头 + 每局不同 | 程序化地图生成 + 随机掉落 + 永久成长 | ⭐⭐⭐⭐ |
| **文字冒险/视觉小说** | 对话选项驱动剧情，轻度或无战斗 | 对话树/分支系统 + 状态标记 + CG展示 | ⭐⭐⭐ |

**推荐首选**：俯视角ARPG（画面效果好）、文字冒险（最快出原型）

---

### 七、竞速 / 体育

**定位**：操作技巧 + 速度感；核心乐趣是刷新记录和超越对手的即时反馈。

| 游戏 | 一句话说明 | 核心机制 | 开发复杂度 |
|------|-----------|---------|-----------|
| **2D 赛车** | 俯视角/侧视角赛车，弯道漂移，争第一 | 车辆物理（转向/加速/漂移）+ 赛道碰撞 | ⭐⭐⭐ |
| **跑步 / 田径小游戏** | 按键节奏控制角色奔跑、跨栏、跳远 | 节奏判定（精确时机）+ 速度积累 | ⭐⭐ |
| **足球 / 篮球迷你游戏** | 简化规则的球类运动小游戏 | 物理抛物线 + 目标判定 + 对手AI | ⭐⭐⭐ |
| **3D 赛车原型** | 第三人称视角的赛车游戏 | 3D 车辆物理 + 赛道建模 + 碰撞体 | ⭐⭐⭐⭐⭐ |

**推荐首选**：2D 赛车（技术难度最适中）、田径小游戏（最快出原型）

---

### 八、解谜密室

**定位**：逻辑推理 + 空间思考；核心乐趣是解开谜题后的智识成就感。

| 游戏 | 一句话说明 | 核心机制 | 开发复杂度 |
|------|-----------|---------|-----------|
| **推箱子（Sokoban）** | 把箱子推到指定位置，不能拉回 | 格子移动 + 碰撞规则 + 关卡验证（死锁检测） | ⭐⭐ |
| **数独 / 填数谜题** | 在格子中填入数字，满足行列块约束 | 约束满足算法 + 合法性验证 + 提示系统 | ⭐⭐⭐ |
| **密室逃脱（点击冒险）** | 在房间里点击物品寻找线索、解开谜题离开 | 场景切换 + 物品交互/组合 + 状态机管理 | ⭐⭐⭐ |
| **光线反射谜题** | 放置镜子/棱镜让光线照到目标点 | 向量反射计算 + 碰撞体 + 谜题验证 | ⭐⭐⭐ |
| **接线/电路谜题** | 连接电路节点让电流从源点到达目标 | 图的连通性算法 + 拖拽交互 + 路径验证 | ⭐⭐⭐ |

**推荐首选**：推箱子（最容易出关卡编辑器）、密室逃脱（体验感最强）

---

### 开发复杂度说明

- ⭐⭐：1-3天可出可玩原型
- ⭐⭐⭐：1周可出可玩原型
- ⭐⭐⭐⭐：2-4周，需要系统设计
- ⭐⭐⭐⭐⭐：1个月以上，需要完整项目规划

## 二、推荐对话话术规范

> 核心原则：**给推荐要给理由，给理由要有主推，每步带出下一步**

### 展示类型菜单时

菜单前先说一句引导语，不要直接扔选项：

> "游戏方向挺多的，我先给你列一下，你看哪个感觉对——不确定也没关系，选了之后我帮你具体推荐合适的游戏。"

菜单每项附上氛围描述（见上方「类型菜单」格式）。

---

### 推荐具体游戏时

❌ 不要只列名字和⭐数

✅ 说清楚三件事：**有意思在哪 + 主要坑是什么 + 大概要多久**，末尾给一个主推建议：

> "这几款都挺典型的，我说一下各自有意思在哪：
>
> **扫雷**——逻辑推理，靠数字推断地雷位置。别看规则简单，有个小细节很多人做扫雷会忘：「第一次点击必须安全」——不处理这个，玩家第一手就踩雷体验直接崩。难度⭐⭐，3天内可以出完整可玩版本。
>
> **贪吃蛇**——蛇越长越难操控。代码是最简单的，想做出差异化需要在规则或视觉上加点料。难度⭐⭐，下午可以出来。
>
> 你手头大概有多少时间？想快点出成果我建议先做贪吃蛇或扫雷。"

---

### 用户选定具体游戏后

1. 说核心机制（3-5句，设计层视角）
2. 给一条主动建议：「如果是我，我会先把XX做出来」
3. 问一个最关键的问题，引导进入框架设计

示例（用户选了扫雷）：
> "扫雷核心就三块：棋盘数组、地雷生成、周围地雷数计算。最容易踩的坑是「点开空白自动展开」这块，要用递归或BFS，写不对玩起来会很别扭。
>
> 我建议先把棋盘规格定了——经典有三档：简单9×9/10颗雷，中等16×16/40颗雷，困难30×16/99颗雷。你打算做固定难度还是让玩家选？"

---

### 特殊情况处理

**用户说「不知道做什么」**：
> "不确定很正常，我问你一个问题帮你找方向：你之前有没有特别喜欢或者印象深的游戏？不用是最近的，随便说一个——我来帮你找那种感觉对应的游戏类型，然后推荐能做出来的版本。"

**用户说复合类型**（如「带RPG的塔防」）：
> "这个组合可以做，主框架建议先定塔防——核心循环跑通之后，RPG元素可以叠在上面。先把塔防基础做出来，还是聊聊RPG那层要加哪些东西？"

**用户直接说游戏名**（如「我想做扫雷」）：跳过菜单，直接给核心机制介绍+建议。

---

### 对话反模式（绝对禁止）

| 反模式 | 禁止示例 | 问题所在 |
|--------|---------|---------|
| 万能开场白 | "好的！""当然！""没问题！" | 没有信息量，纯粹占位 |
| 只给答案不给理由 | "你可以做扫雷。" | 用户不知道为什么 |
| 废话总结段 | "以上就是我的建议，希望对你有帮助！" | 结尾什么都没说 |
| 问题轰炸 | "请问你用什么引擎？目标平台？团队规模？预算？" | 一次只问最关键的一个 |
| 对称废话 | "可以，也可以不可以，取决于你的情况" | 没有立场，没有帮助 |
