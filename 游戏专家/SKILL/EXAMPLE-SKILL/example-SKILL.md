# SKILL.md - kling-image-gen（游戏概念图生成）

> **示例 Skill** — 当需要生成游戏视觉参考图、UI概念图、角色设计图、关卡意境图时启用

## 能力描述

调用可灵 Omni 图像生成接口（kling-image-o1 / kling-v3-omni），根据文字描述生成游戏概念图、UI风格参考、角色设计草图、关卡视觉意境图等。在游戏开发场景中，帮助开发者快速建立视觉参考，为美术方向讨论提供具体素材而非空谈。

支持多种画面比例（16:9/9:16/1:1等），单次可生成1~9张，分辨率最高2K。

**工作流：提交任务 → 轮询结果（约7秒/次）→ 获取图片URL**

## 触发场景

| 场景 | 触发条件 | 示例 |
|------|----------|------|
| 游戏UI风格参考图 | 讨论UI风格时，需要快速生成视觉样本 | "生成一张像素风塔防游戏的HUD参考图" |
| 关卡意境图 | 描述关卡氛围，生成视觉参考 | "生成一张俯视角扫雷游戏界面风格图" |
| 角色/单位设计草图 | 讨论角色视觉风格时快速出图 | "生成大鱼吃小鱼游戏里的BOSS鱼概念图" |
| 游戏封面/宣传图 | 项目需要宣传素材 | "生成一张横版塔防游戏宣传图，16:9" |
| 多方案视觉对比 | 同时生成多张不同风格的参考图 | "生成3张不同风格的扫雷界面参考图" |

## 调用规范

- **默认模型**：`kling-image-o1`（质量优先）；快速出图用 `kling-v3-omni`
- **调用前说明**：告知用户"我来生成一张参考图，大约需要30~60秒"
- **提示词原则**：在用户描述基础上加入游戏开发专用细节（分辨率要求、风格词、用途）
- **画面比例**：默认 `1:1`（UI参考）；场景图用 `16:9`；手机游戏用 `9:16`
- **生成数量**：单次建议1~3张，避免过多图片干扰决策

## 提示词示例（游戏开发场景）

```
塔防游戏HUD界面参考图，像素风格，顶部显示金币和生命值，
底部植物/单位选择栏，简洁扁平设计，配色以绿色和棕色为主，游戏界面截图风格
```

```
俯视角扫雷游戏棋盘，经典Windows风格现代化设计，
9x9格子，数字1-8颜色区分，灰色未翻格，白色翻开格，清晰易读
```


## 生成结果处理

- 图片 URL 有效期 **30天**，需及时转存（可调用 `image-upload` 存至 Supabase Storage）
- 生成结果标注"此为AI生成的风格参考，不代表最终美术方案"
- 如生成效果不符合预期，提供优化后的提示词建议再次生成

## 硬性约束

- 不生成含版权角色（不生成"植物大战僵尸官方角色"等有版权内容）
- 生成图仅作视觉参考，不作为最终美术资产使用
- 每次生成前确认用户的具体需求（风格/用途/比例），不凭空猜测
